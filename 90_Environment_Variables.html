Environment Variables

Environment Variables are configuration values stored outside the application code. They help manage sensitive information such as API keys, database credentials, and application settings securely.

Application → Reads Environment Variables → Uses Configuration

Example .env File

PORT=3000
DB_HOST=localhost
DB_USER=admin
DB_PASSWORD=secret123
API_KEY=your_api_key
JWT_SECRET=my_secret_key

Node.js Example

require('dotenv').config();

console.log("Server Port:", process.env.PORT);
console.log("Database Host:", process.env.DB_HOST);

Installing dotenv

npm install dotenv

Common Environment Variables

Variable Purpose
PORT Application port number
DB_HOST Database server address
DB_USER Database username
DB_PASSWORD Database password
API_KEY Third-party API access key
JWT_SECRET Authentication token secret

Benefits

  • Improves security by separating secrets from code.
  • Supports multiple environments (Development, Testing, Production).
  • Makes configuration easier to manage.
  • Reduces hard-coded values.
  • Simplifies deployment and maintenance.

Best Practices

  • Never commit .env files to public repositories.
  • Use strong secret keys and passwords.
  • Store production secrets securely.
  • Validate required environment variables at startup.
  • Use different configurations for each environment.
