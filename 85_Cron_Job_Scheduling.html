Cron Job Scheduling

Cron Jobs are automated tasks that run at scheduled times on a server. They are commonly used for backups, report generation, email sending, database cleanup, and other recurring tasks.

How Cron Works

Scheduler → Executes Task → Logs Result

Cron Expression Format

* * * * * command
| | | | |
| | | | └── Day of Week (0-7)
| | | └──── Month (1-12)
| | └────── Day of Month (1-31)
| └──────── Hour (0-23)
└────────── Minute (0-59)

Examples

Cron Expression Description
* * * * * Every minute
0 * * * * Every hour
0 0 * * * Every day at midnight
0 9 * * 1 Every Monday at 9 AM

Linux Cron Example

crontab -e

# Run backup every day at 2 AM
0 2 * * * /home/user/backup.sh

Node.js Example

const cron = require('node-cron');

cron.schedule('0 9 * * *', () => {
    console.log('Task runs every day at 9 AM');
});

Common Use Cases

  • Database backups
  • Sending emails and notifications
  • Cleaning temporary files
  • Generating reports
  • Synchronizing data
  • Monitoring system health
